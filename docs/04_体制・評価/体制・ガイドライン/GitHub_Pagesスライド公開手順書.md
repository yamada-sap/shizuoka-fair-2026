# 📖 GitHub Pages スマホ用スライド自動公開 設定手順書

本リポジトリ内の `output/` フォルダにあるMarpスライドのHTMLやPDFファイルを、スマホのブラウザからURLで直接アクセスし、スライドショーのように左右スワイプで閲覧するための自動公開（GitHub Pages）の設定ガイドです。

---

## 🛠️ ステップ 1. ワークフロー（YAML）の確認

リポジトリ内に自動デプロイ用のワークフロー設定ファイル [pages_deploy.yml](file:///Users/sap220701/Desktop/産業フェア_ローカル/.github/workflows/pages_deploy.yml) が配置されていることを確認します。
これにより、`output/` 内のHTMLファイルが更新（`push`）されるたびに、自動的にWebサイトとしてデプロイされます。

---

## 🔓 ステップ 2. GitHubリポジトリの設定（初回のみ）

自動デプロイのアクションが正しく動作するよう、GitHubリポジトリの設定を変更します。

1. ブラウザでリポジトリを開き、**「Settings」** タブをクリックします。
2. 左メニューの **「Pages」** をクリックします。
3. **「Build and deployment」** セクションの **「Source」** ドロップダウンメニューを確認します。
4. デフォルトの `Deploy from a branch` から、**「GitHub Actions」** に変更します。
   * ※ これにより、先ほど配置した `pages_deploy.yml` からの自動デプロイ要求を受け付けるようになります。

---

## 🚀 ステップ 3. 自動デプロイの実行とアクセス

1. ローカルでスライドHTML（例: `test_preview.html`）をビルドして `output/` に保存します。
2. `git add`, `git commit`, `git push` を実行します。
3. リポジトリの **「Actions」** タブで `Deploy Slides to GitHub Pages` ジョブが走り、約30秒〜1分で完了します。
4. 完了すると、以下のURLでスマホやPCのブラウザからスライドショー形式でアクセス可能になります。
   * **URLの形式**: `https://<あなたのGitHubユーザー名>.github.io/<リポジトリ名>/<ファイル名>.html`
   * **例**: `https://<ユーザー名>.github.io/産業フェア_ローカル/test_preview.html`

---

## 📱 スマホでのスライドショーのプレビュー方法
* スマホのSafariやChromeで上記URLを開くと、**画面を左右にスワイプ（フリック）するだけ**で、まるでパワーポイントの発表用スライドのようにスムーズにページをめくってプレビューできます。
* 画面左下（または右下）の矢印ボタンをタップしてめくることも可能です。
