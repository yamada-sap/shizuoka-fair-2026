# 📖 Notion 🔄 GitHub 双方向同期システム セットアップ手順書

本イベントプロジェクトのMarkdown資料群と、Notionの各ページを双方向に同期するための設定ガイドです。
これにより、AIエディタ（ローカル）で作成した計画書や討論ログを、移動中にスマートフォンのNotionアプリからいつでも検索・閲覧・更新できるようになります。

---

## 🛠️ ステップ 1. Notion APIトークン（インテグレーション）の作成

1. ブラウザで [Notionマイインテグレーション](https://www.notion.so/my-integrations) にアクセスします（Notionにログインした状態で行ってください）。
2. **「+ 新しいインテグレーション」**ボタンをクリックします。
3. 以下の項目を設定します。
   * **名前**: `GitHub-Marp-Sync`（任意）
   * **関連ワークスペース**: 同期したいページがあるワークスペースを選択
   * **機能**: 「コンテンツを読み取る」「コンテンツを更新する」「コンテンツを挿入する」にチェックが入っていることを確認（デフォルトでON）
4. **「送信」**をクリックして作成を完了します。
5. 作成後、画面に表示される**「内部インテグレーションシークレット」**（例: `secret_xxxxxx...`）をコピーしてメモしておきます（これが `NOTION_TOKEN` になります）。

---

## 🔓 ステップ 2. 同期対象のNotionページにインテグレーションを許可する

Notionのセキュリティ上、作成したインテグレーションがページにアクセスできるように明示的な「コネクト許可」が必要です。

1. 同期したいNotionの**「一番親となるページ」**（または各同期対象ページ）をブラウザまたはアプリで開きます。
2. 画面右上にある **「・・・」（もっと見るボタン）** をクリックします。
3. ポップアップ下部にある **「コネクトの追加（または コネクト）」** を探し、ステップ1で作成したインテグレーション名（`GitHub-Marp-Sync`）を選択して接続を許可します。
   * ※ 親ページで一度許可すれば、その配下の子ページにも自動的にアクセス権が継承されます。

---

## 🔑 ステップ 3. GitHubのリポジトリにSecretsを登録する

GitHub上で同期スクリプトが安全に実行されるよう、認証情報と同期マップを登録します。

1. GitHub上のリポジトリを開き、**「Settings」** タブ ➔ 左メニューの **「Secrets and variables」** ➔ **「Actions」** を開きます。
2. **「New repository secret」** ボタンをクリックし、以下の2つのシークレットを登録します。

### ① `NOTION_TOKEN`
* **Name**: `NOTION_TOKEN`
* **Value**: ステップ1でコピーした `secret_xxxxxx...` をそのまま貼り付けます。

### ② `NOTION_PAGE_MAP`
* **Name**: `NOTION_PAGE_MAP`
* **Value**: 同期したい「ローカルファイルパス」と「NotionのPage ID」のマッピングを、以下の形式の**JSON形式**で記述します（ダブルクォーテーションを厳密に使用してください）。

```json
{
  "README.md": "3c7f98e7ba454b6fb89ef062a4d95221",
  "docs/04_体制・評価/プロジェクト全体タスク管理簿.md": "a8b9c1d2e3f4g5h6i7j8k9l0m1n2o3p4"
}
```

> [!TIP]
> **NotionのPage IDの確認方法**
> 対象のNotionページを開き、ブラウザのURLを確認します。
> 例: `https://www.notion.so/myworkspace/My-Page-Title-3c7f98e7ba454b6fb89ef062a4d95221`
> URLの末尾（タイトルより後ろ）にある **32桁の英数字（ハイフンを除く）** がPage IDです。

## 🚀 ステップ 4. 同期動作のテスト

### 1. ローカル ➔ Notion（自動同期の確認）
1. ローカルで `README.md` などのMarkdownファイルを適当に編集します。
2. 変更を `git commit` し、GitHubの `main` ブランチへ `git push` します。
3. GitHubリポジトリの **「Actions」** タブで `Notion Bilateral Sync` ワークフローが起動し、数秒でNotion側のページ内容が書き換わることを確認します。

### 2. Notion ➔ ローカル（手動同期の確認）
1. 外出先などで、スマートフォンのNotionアプリから同期対象のページの一部（例: TODOリストのチェックマークを `[x]` にするなど）を編集します。
2. PCに戻るか、あるいはGitHub Actionsの画面を開きます。
3. **「Actions」** タブ ➔ 左側のリストから **「Notion Bilateral Sync」** ワークフローを選択します。
4. 画面右側にある **「Run workflow」** ボタンをクリックし、実行します。
5. ワークフローが完了すると、Notion側の変更内容がMarkdownに自動変換され、リポジトリに `chore: sync content from Notion` というコミットでプッシュされ、ローカルへ `pull` すれば同期が完了します。
